# FormTextArea

`FormTextArea` is a customizable textarea component that standardizes `textarea` behavior and styling while ensuring a consistent user experience.

## Import

```tsx
import { FormTextArea } from 'piekit'
```

## Basic usage

<FormTextArea label='Name' placeholder='Enter name' />
<br />

```tsx
<FormTextArea label='Name' placeholder='Enter name' />
```

## Props

### Label
Default value: `undefined`\
Type: `string`\
Description: The `label` prop will enable the label on top of the textarea:


### Disabled

Default value: `false`\
Type: `boolean`\
Description: If `disabled` prop is set to `true`, the textarea will be disabled.

<FormTextArea placeholder='Enter name' disabled />
<br />

```tsx
<FormTextArea placeholder='Enter name' disabled={true} />
```

### Error state

Default value: `false`\
Type: `boolean`\
Description: If `hasError` prop is set to `true`, the textarea appearance will change to `error` variant.

<FormTextArea
    label='Name'
    placeholder='Enter name'
    hasError={true}
    errorMessages={['Field required']} />
<br />

```tsx
<FormTextArea
    label='Name'
    placeholder='Enter name'
    hasError={true}
    errorMessages={['Field required']} />
```

### Error messages

Default value: `[]`\
Type: `string[]`\
Description: If `errorMessages` prop is being set, error messages will be shown. `hasError` have to be set to `true`

### Test automation tag

To assist with automated testing, add a `testId` attribute to the `FormTextArea`.

```tsx
<FormTextArea
    testId='ta-name'
    label='Name'
    placeholder='Enter name'
    hasError={true}
    errorMessages={['Field required']} />
```

Rendered as:

```html
<div data-testid="ta-name-form">
    <label data-testid="ta-name-label">Name</label>
    <textarea placeholder="Enter name" data-testid="ta-name-textarea"></textarea>
    <p data-testid="ta-name-error-msg-0">Field required</p>
</div>
```

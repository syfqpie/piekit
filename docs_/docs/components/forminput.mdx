# FormInput

`FormInput` is a customizable input component that standardizes `input` behavior and styling while ensuring a consistent user experience.

## Import

```tsx
import { FormInput } from 'piekit'
```

## Basic usage

<FormInput label='Name' placeholder='Enter name' />
<br />

```tsx
<FormInput label='Name' placeholder='Enter name' />
```

## Props

### Label
Default value: `undefined`\
Type: `string`\
Description: The `label` prop will enable the label on top of the input:


### Disabled

Default value: `false`\
Type: `boolean`\
Description: If `disabled` prop is set to `true`, the input will be disabled.

<FormInput placeholder='Enter name' disabled />
<br />

```tsx
<FormInput placeholder='Enter name' disabled={true} />
```

### Error state

Default value: `false`\
Type: `boolean`\
Description: If `hasError` prop is set to `true`, the input appearance will change to `error` variant.

<FormInput
    label='Name'
    placeholder='Enter name'
    hasError={true}
    errorMessages={['Field required']} />
<br />

```tsx
<FormInput
    label='Name'
    placeholder='Enter name'
    hasError={true}
    errorMessages={['Field required']} />
```

### Error messages

Default value: `[]`\
Type: `string[]`\
Description: If `errorMessages` prop is being set, error messages will be shown. `hasError` have to be set to `true`

### Test automation tag

To assist with automated testing, add a `testId` attribute to the `FormInput`.

```tsx
<FormInput
    testId='ta-name'
    label='Name'
    placeholder='Enter name'
    hasError={true}
    errorMessages={['Field required']} />
```

Rendered as:

```html
<div data-testid="ta-name-form">
    <label data-testid="ta-name-label">Name</label>
    <input placeholder="Enter name" data-testid="ta-name-input">
    <p data-testid="ta-name-error-msg-0">Field required</p>
</div>
```

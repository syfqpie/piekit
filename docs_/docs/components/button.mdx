# Button

The `Button` component is a customizable button element that extends the default HTML `<button>` element. It inherits all the default properties of a `<button>` and adds custom functionality through additional props.

## Import

```tsx
import { Button } from 'piekit'
```

## Basic usage

<Button size="sm" variant="primary" onClick={() => alert('Button clicked!')}> Click me </Button>
<br />
<br />

```tsx
<Button
    size="sm"
    variant="primary"
    onClick={() => alert('Button clicked!')}>
  Click me
</Button>
```

## Props

### Variant

Default value: `primary`\
Type: `primary` | `success` | `error`\
Description: The `variant` prop defines the button's visual style. It can be one of the following:
- `primary`: The default button style.
- `success`: A style used for positive actions, often with a green background or border.
- `error`: A style that indicates a negative or destructive action, often with a red background or border.

<Button> Primary </Button>
<Button variant='success'> Success </Button>
<Button variant='error'> Error </Button>
<br />
<br />

```tsx
<Button>
    Primary 
</Button>

<Button variant='success'>
    Success 
</Button>

<Button variant='error'>
    Error 
</Button>
```

### Appearance

Default value: `solid`\
Type: `solid` | `outline`\
Description: The `appearance` prop defines the button's style, determining whether it has a solid background or an outline:
- `solid`: The default appearance, with a solid background color.
- `outline`: A button with a border and transparent background.

<Button appearance='outline'> Primary </Button>
<Button appearance='outline' variant='success'> Success </Button>
<Button appearance='outline' variant='error'> Error </Button>
<br />
<br />

```tsx
<Button appearance='outline'>
    Primary outline 
</Button>

<Button appearance='outline' variant='success'>
    Success outline 
</Button>

<Button appearance='outline' variant='error'>
    Error outline
</Button> 
```

### Size

Default Value: `md`\
Type: `xs`, `sm`, `md`, `lg`\
Description: The `size` prop controls the size of the button. It can be one of the following:
- `xs`: An extra small button
- `sm`: A smaller button, suitable for more compact UI layouts.
- `md`: A medium button which is the default button size.
- `lg`: A larger button for more prominent actions.

<Button size='lg'> Large </Button>
<Button> Medium </Button>
<Button size='sm'> Small </Button>
<Button size='xs'> Extra small </Button>
<br />
<br />

```tsx
<Button size='lg'>
    Large
</Button>

<Button>
    Medium
</Button>

<Button size='sm'>
    Small
</Button>

<Button size='xs'>
    Extra small
</Button>
```

### Disabled

Default value: `false`\
Type: `boolean`\
Description: If `disabled` prop is set to `true`, the button will be disabled and will not trigger any click events.

<Button disabled> Click me </Button>
<br />
<br />

```tsx
<Button disabled>
    Click me
</Button>

<Button disabled={true}>
    Click me
</Button>
```

### Full

Default value: `false`\
Type: `boolean`\
Description: If `full` prop is set to `true`, the button will stretch to fill the full width of its container.

<Button full> Click me </Button>
<br />
<br />

```tsx
<Button full={true}>
    Solid
</Button>
```

### Alignment

Default value: `center`\
Type: `start`, `center`, `end`\
Description: The `alignment` prop controls the alignment of the content inside the button:
- `start`: Aligns the content to the left of the button.
- `center`: Centers the content (default behavior).
- `end`: Aligns the content to the right of the button.

<div style={{ display: 'flex', flexDirection: 'column', gap: '0.5em' }}>
    <Button full={true} alignment='start'> Left </Button>
    <Button full={true} alignment='center'>	Center </Button>
    <Button full={true} alignment='end'> Right </Button>
</div>
<br />
<br />

```tsx
<Button full={true} alignment='start'>
    Left
</Button>

<Button full={true} alignment='center'>
    Center
</Button>

<Button full={true} alignment='end'>
    Right
</Button>
```

### Test automation tag

To assist with automated testing, add a `testId` attribute to the `Button`.

```tsx
<Button testId='button-ta'>
    Click me
</Button>
```

Rendered as:

```html
<button data-testid="button-ta">
    Click me
</button>
```

## Inherited Default Props

The `Button` component inherits the standard HTML `<button>` attributes, including but not limited to:

- `type` (default: `button`): Specifies the button type (e.g., `submit`, `reset`, `button`).
- `name`: Defines the name of the button.
- `value`: Specifies the value associated with the button in forms.
- `aria-*`: Any `aria` attributes for accessibility (e.g., `aria-label`, `aria-hidden`).

This allows you to pass standard button props seamlessly into the `Button` component.
